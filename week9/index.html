    I found a website that better helped me understand between local storage, session storage, and cookies: www.xenonstack.com

    If my understanding is correct, the differences between local storage, session storage, and cookies are as follows:

    • Local Storage: Data with key:value pairs (Storage Capacity: 10MB), that is saved until it is manually deleted. Data does not expire and does not end when user closes the window or tab. Does not exchange with the server.
    • Session Storage: Data with key:value pairs (Storage Capacity: 5MB), that is saved only as long as the tab remains open. When tab closes, data is terminated. Does not exchange with the server.
    • Cookie: Data that stores one key:value pair at a time (Storage Capacity: 5KB) and is saved only as long as the tab remains open. When the tab closes, the data is terminated. Exchanges with the server upon any request.

      I didn't know this, but there are actually two diffent types of Cookies.
      • Session Cookies: They expire when the tab closes.
      • Persistent Cookies: They can be set to expire on a specific date or for a specified length of time.
      Cookies were also the only way to store data before HTML5. This must have been a game changer.
